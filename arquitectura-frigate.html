Sistema de detección de ladridos

Arquitectura técnica · Synology DS225+ · Junio 2026

Esquema de la arquitectura

Cómo funciona paso a paso

1
La cámara transmite audio por RTSP La Tapo C210 emite un flujo de vídeo y audio continuo por el protocolo RTSP en el puerto 554. Este flujo viaja por la red local de tu casa.
2
go2rtc gestiona la conexión al stream go2rtc (incluido dentro de Frigate) se conecta a la cámara y gestiona el stream de forma eficiente. Frigate le delega esta tarea para evitar problemas de permisos.
3
Frigate analiza el audio con IA Frigate extrae el audio del stream y lo pasa por un modelo de inteligencia artificial (TensorFlow Lite) entrenado para reconocer sonidos. Está configurado para escuchar específicamente ladridos de perro ("bark") con un umbral de confianza del 80%.
4
Frigate publica el evento en MQTT Cuando detecta un ladrido, Frigate publica un mensaje en el topic frigate/sala_tapo/audio/bark del broker Mosquitto.
5
El puente Python recibe el evento bridge.py está suscrito a ese topic MQTT. En cuanto llega el mensaje, lo lee e identifica que es un evento nuevo ("type": "new"). Tiene un cooldown de 10 segundos para evitar spam.
6
El puente dispara el webhook bridge.py hace una petición HTTP POST a la URL de Surveillance Station con el token de autenticación. Surveillance Station recibe la llamada y genera una notificación.
7
Recibes la notificación en el móvil La app DS cam recibe la notificación push de Surveillance Station y te avisa en tu móvil, estés donde estés.

Tecnologías utilizadas

Detección de audio

Frigate NVR

Sistema de videovigilancia de código abierto especializado en detección de objetos y audio mediante IA. Usa TensorFlow Lite para clasificar sonidos en tiempo real. Es el cerebro de todo el sistema.

Gestión de streams

go2rtc

Servidor de streams multimedia integrado en Frigate. Gestiona la conexión RTSP con la cámara y redistribuye el audio/vídeo internamente. Resuelve los problemas de compatibilidad con cámaras Tapo.

Mensajería

Mosquitto (MQTT)

Broker de mensajería ligero para dispositivos IoT. Actúa de "central de comunicaciones": Frigate le envía eventos y el puente Python los recibe. El protocolo MQTT está diseñado para ser eficiente y fiable en redes locales.

Integración

bridge.py (Python)

Microservicio puente escrito en Python. Su único trabajo es escuchar el topic MQTT de ladridos y disparar el webhook de Surveillance Station. Pesa ~19MB de RAM y tiene reconexión automática.

Notificaciones

Surveillance Station

Sistema de gestión de cámaras de Synology. Recibe el webhook y envía la notificación push a la app DS cam en tu móvil. Ya tenías este servicio funcionando con las cámaras Tapo.

Infraestructura

Docker / Container Manager

Plataforma de contenedores que ejecuta los tres servicios (Frigate, Mosquitto, bridge.py) de forma aislada y fiable. El Container Manager de Synology proporciona la interfaz gráfica para gestionarlos.

Archivos clave del sistema

/volume1/docker/frigate/
docker-compose.yml

Define y orquesta los tres contenedores: Frigate, Mosquitto y el puente. Es el archivo principal de la instalación.

/volume1/docker/frigate/config/
config.yml

Configuración de Frigate: qué cámara escuchar, qué sonidos detectar, umbral de confianza y conexión MQTT.

/volume1/docker/frigate/mosquitto/
mosquitto.conf

Configuración del broker MQTT: puerto, permisos de conexión anónima (red local) y persistencia de mensajes.

/volume1/docker/frigate/bridge/
bridge.py

Script Python del puente. Lee el topic MQTT y dispara el webhook de Surveillance Station con cooldown configurable.

Configuración de Frigate (config.yml)

# Conexión al broker MQTT (Mosquitto en la misma red Docker) mqtt: enabled: true host: 192.168.1.162 # IP del Synology port: 1883 # Detección de audio con IA audio: enabled: true listen: - bark # Escuchar ladridos de perro filters: bark: threshold: 0.80 # 80% de confianza mínima # go2rtc gestiona el stream para evitar problemas con Tapo go2rtc: streams: sala_tapo: - rtsp://usuario:contraseña@192.168.1.167:554/stream1 # Frigate se conecta a go2rtc localmente cameras: sala_tapo: ffmpeg: inputs: - path: rtsp://127.0.0.1:8554/sala_tapo roles: - audio
¿Por qué go2rtc? Las cámaras Tapo tienen restricciones de autenticación RTSP que generan errores "Operation not permitted" cuando ffmpeg intenta conectarse directamente. go2rtc actúa como intermediario y resuelve este problema.
